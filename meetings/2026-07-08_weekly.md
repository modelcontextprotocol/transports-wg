# 2026-07-08 MCP: Transport Working Group (Weekly)

## TL;DR
The Transport Working Group aligned on major architectural updates regarding
tool capabilities, agreeing to focus exclusively on client-side tool annotations
to simplify protocol complexity. The team also finalized a roadmap for streaming
and internationalization, discussed SDK pagination and caching mechanisms, and
explored rescheduling the weekly meeting to better accommodate international
members.

## Detailed Summary

### Technical Protocol Updates & Caching
* The team discussed caching mechanisms for tool lists using unique version
  keys.
* Shaun Smith and the group will work on drafting explicit guidance regarding
  what constitutes a cache key for implementations to follow to ensure
  consistent client-server coordination.
* In the Software Development Kit (SDK), pagination issues were identified and
  clarified as distinct from current caching pull requests.

### Refining Tool Capability Models
* The group discussed a decision regarding capability model design, discussing a
  focus on client-side tool annotations vs server-side filtering.
* Client-side annotation simplifies the overall protocol implementation and
  avoids unnecessary caching complexity for proxies.
* Gabriel Zimmerman will develop the next version of the tool capabilities
  proposal and outline use cases/examples where elicitation capabilities impact
  tool calls.

### Roadmap Planning & Meeting Schedule
* The working group discussed the roadmap covering upcoming goals like streaming
  capabilities and internationalization.
* The July specification release remains on track, with preview versions already
  available for testing.
* To improve collaboration across differing time zones, the group debated
  potential rescheduling, including a possible move to Friday mornings.
