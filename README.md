# MCP Transports Working Group

Welcome to the **MCP Transports Working Group** — a collaborative community focused on improving the usability of transports in the [Model Context Protocol (MCP)](https://modelcontextprotocol.io), including scalability, reliability, and semantics.

## Mission

The Transports Working Group coordinates community stakeholders to:

- **Improve transport usability** across MCP implementations
- **Define scalability patterns** for connection pooling, load distribution, and resource management
- **Establish reliability standards** for error handling, reconnection, and message delivery guarantees
- **Clarify transport semantics** for protocol behavior, message ordering, and state management
- **Drive interoperability** across platforms and implementations

> **See the [CHARTER](./CHARTER.md) for detailed scope and processes.**

## Getting Involved

We welcome contributions from anyone interested in MCP transports! Here is how you can get involved:

1. **Join the community** — Connect on the `#transports-wg` channel in [MCP Discord](https://discord.com/invite/6CSzBmMkjX).
2. **Attend meetings** — Join our weekly calls (find the next meeting at [https://meet.modelcontextprotocol.io/](https://meet.modelcontextprotocol.io/)). No approval needed!
3. **Contribute** — Open issues to propose ideas or submit PRs for improvements.

Please review our **[CONTRIBUTING.md](./CONTRIBUTING.md)** guide for detailed guidelines on how to participate.

All participants must abide by the **[MCP Code of Conduct](https://github.com/modelcontextprotocol/.github/blob/main/CODE_OF_CONDUCT.md)** and adhere to the **[MCP Antitrust Policy](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/ANTITRUST.md)**.

_See [GOVERNANCE.md](./GOVERNANCE.md) for current members and governance roles._

## Current Focus Areas

Our work focuses on:

1. **Scalability** — Connection pooling, load distribution, resource management
2. **Reliability** — Error handling, reconnection strategies, message delivery guarantees
3. **Transport Semantics** — Protocol behavior, message ordering, state management
